
# Концептуальная основа алгоритмов

Согласно учебной литературе, решёточные методы базируются на двух ключевых этапах: столкновение и перенос. 
В отличие от традиционных вычислительных методов гидродинамики (например, метода конечных разностей для уравнений Навье-Стокса), здесь моделируется не движение сплошной среды, а эволюция функции распределения частиц на фиксированной решётке.

# Алгоритм для решёточных газов (Клеточно-автоматный подход)

Этот метод чаще всего иллюстрирует переход от дискретной микродинамики к макроскопическому поведению.

- Инициализация конфигурации: Задаётся решётка (например, гексагональная FHP, чтобы обеспечить изотропию макроскопических уравнений).

- Шаг 1: Локальное столкновение: Применяется правило столкновения (таблица переходов), которое сохраняет массу и импульс.

- Шаг 2: Перенос (Propagation): Все частицы синхронно перемещаются в соседние узлы по своим направлениям.

- Граничные условия: Реализуются через отражение: частица, ударяющаяся о стенку, меняет направление на противоположное.

# Алгоритм решёточного уравнения Больцмана (LBM)

В учебниках этот метод представлен как сглаженная и более стабильная версия LGA, где шум усредняется.

1) Дискретизация пространства и скоростей: Выбирается модель решётки (D2Q9, D3Q19), задаются векторы скорости и весовые коэффициенты.

2) Основной цикл (алгоритм «столкновение-перенос»): Вычисление макроскопических переменных: Плотность и скорость.

# Вычислительные особенности (с точки зрения реализации алгоритмов)

В учебной литературе по моделированию на ПК подчёркиваются следующие аспекты реализации:

1) Локальность данных: Главное преимущество — алгоритм коллизии требует только данных текущего узла. Это делает LBM идеальным для параллельных вычислений (MPI, OpenMP, CUDA).

2) Пространственная декомпозиция: Для распределённой памяти расчётная область разбивается на подблоки. Каждый процессор хранит «толстые» границы (ghost layers) для обмена данными после шага переноса.

3) Граничные условия: Основные методы, описываемые в учебниках:

- Bounce-back (отскок): Стандарт для непроницаемых стенок (моделирует прилипание).

- Периодические: Для замыкания области.

- Zou-He: Для задания скорости/давления на входе/выходе.

# Последовательность численного решения

В методических пособиях рекомендуется следующий порядок действий для реализации кода:

Настройка параметров: Размеры сетки, параметр релаксации, константы решётки.
Инициализация: Установка во всей области.
Цикл по времени: Выполнить коллизию во всех узлах (локальная операция), выполнить перенос во всех узлах (сдвиг данных), применить граничные условия (обработка выпадающих направлений), рассчитать макроскопические поля (скорость, давление) для визуализации.
Выход: Сохранение результатов, когда течение выходит на стационарный режим.

# Сравнение LGA и LBM

Для презентации можно использовать следующее сравнение алгоритмов:

1) Переменные: LGA оперирует битами (0 или 1), LBM — вещественными числами (float/double).

2) Коллизия: В LGA — это таблица истинности (квантованная), в LBM — аналитическая формула релаксации.

3) Шум: LGA требует усреднения по времени/пространству (шумный), LBM даёт гладкое решение.

4) Сложность: LGA проще для понимания, LBM чаще используется в инженерных задачах из-за точности.
